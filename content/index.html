Making Games, #0 : Python

games are a common language for humans

I brought small human[0] to the CoderDojo in The Science Gallery. I actually had to supervise the release of hordes of plague-flea-infested rats in a number of South County Dublin suburbs so that I could get tickets. That's the kind of dedication parenthood can teach you. In that CoderDojo, we did the monkey-see, monkey-do approach to programming - grab a working piece of code, and twitch it about, learning by your mistakes. NOT STRUCTURED. But just to show that YOU CAN CHANGE THINGS. You have power. With power comes confidence. We started off with a game. It was a text game with two caves, one of which has a dragon in. Small human[0] was unimpressed. So we added another dragon. But it was still text. So I downloaded pygame and researched frantically while small human[0] was put to work doing 8-bit art for sprite sheets. At the end of the session, we had an admittedly shite game with shite but personally hand-crafted graphics for achievement. I was ready to smash everything in sight with hammers. My teeth were ground down to nubs. I am not patient. But I had one level of motivation - making games. This shouldn't have been a surprise. That's exactly what I wanted to do too. And having visualized a game, I then learned how to make it. Back in that antediluvian period that meaned learning your local computer assembly language to do smooth bitwise screen scrolling. So you learned more, and found out how much more you could do. That's Games. They are a language for humans. While I was on the train home with small human[0], we designed an 8-bit 2D spritey game with a Rugby theme. And small human[0] uttered the fateful phrase "ok - how do I share my game with my friends?" Getting his buddies to install Python wasn't going to work for this I'm afraid. Ejecta

all of the DOM you need and no more

Ejecta is like a Browser without the Browser. It's specially crafted for Games and Animations. It has no DIVs, no Tables, no Forms – only Canvas and Audio elements. This focus makes it faster. JavaScript code is executed directly by a JavaScript VM (JavaScriptCore), the HTML5 Canvas API is implemented in native code with OpenGL, Audio is implemented with OpenAL. Several other APIs (touch, accelerometer, localStorage) behave like those in a real browser.

Three.js

level up to 3d

WOW. Abstracts away the headaches of getting going with 3D in the browser. With it you can create cameras, objects, lights, materials and more, and you have a choice of renderer, which means you can decide if you want your scene to be drawn using HTML 5's canvas, WebGL or SVG. This has turned out to be the sweet spot for small human[0..1]. Check out the examples here oscar: wrapped a jpg around a sphere geometry tiernan: bouncing camera This is totally the right thing for them - instant results. Shareable in browser. Canvas and WebGL renderers. The WebGL breaks my laptop. Of course, given the internet, there are things that are built on this. Let's talk about Voxels.

voxel.js

make your own voxel worlds

ERMAHGERD. voxel.js is a collection of projects that make it easier than ever to create 3D voxel games like Minecraft all in the browser. It was written by @maxogden and @substack, two non-game developers that want to make voxel games easy, fun and modular. If you find bugs it is because we don't really know what we are doing. The good news is that 100% of the code is on Github so we hope you will fork it. Ok so look at what these guys have built or caused to be built here. Motivation is all about free. Sticks to the unixy principles of tiny tight focussed modules, which is a core factor of the node.js style of building systems. if humans[0..1] saw this, then they would just disappear into it :)
